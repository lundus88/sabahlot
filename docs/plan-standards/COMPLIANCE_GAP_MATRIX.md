# SabahLot Compliance Gap Matrix

Status: Draft QA/QC matrix  
Basis: Audit 1 and Audit 2 findings from code review and compliance review.  
Compliance position: No compliance claim is made for JTU, JUPEM, JKR, PBT, or any authority.

## Severity Values

| Severity | Meaning |
|---|---|
| Critical | Can cause wrong coordinates, wrong legal/technical interpretation, invalid output, or blocked release. |
| High | Significant compliance or QA risk requiring official verification before implementation. |
| Medium | Standardization gap affecting consistency, clarity, or review quality. |
| Low | Documentation, polish, or low-impact control gap. |

## Main Gaps

- Official documents are not available in this repository.
- No compliance claim can be made for JTU, JUPEM, JKR, or PBT.
- CRS, datum, unit, source CRS, output CRS, and transformation record are not fully controlled.
- DXF output is the highest-risk technical gap because it writes latitude/longitude degree values instead of meter-based projected coordinates.
- Graphic standards exist only as code behavior, not as verified plan standards.
- Legend, revision, prepared-by, checked-by, and official template rules remain unverified.
