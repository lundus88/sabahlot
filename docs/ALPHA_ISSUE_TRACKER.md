# SabahLot Alpha Issue Tracker

Purpose: record and classify issues reported during Controlled Alpha testing.

## Issue Severity

| Severity | Meaning |
|---|---|
| Critical | App crash, data loss, serious wrong output, privacy risk |
| Major | Main function failed, import/export failed, save/load failed |
| Minor | UI issue, wording issue, small layout problem |
| Suggestion | User idea or improvement request |

## Issue Status

| Status | Meaning |
|---|---|
| New | Newly reported |
| Reviewing | Being checked |
| Fixed | Fixed and ready for retest |
| Deferred | Accepted but moved to future sprint |
| Not a bug | Expected behaviour or user misunderstanding |

## Priority Rules

1. Critical issues must be fixed before wider release.
2. Major issues must be reviewed before public launch.
3. Minor issues can be grouped into a polish sprint.
4. Suggestions should be recorded but not mixed with bug fixes.
5. Any issue involving boundary, NCR, waris, dispute, ownership, or document sensitivity must be flagged for manual review.
