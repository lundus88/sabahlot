# SabahLot Alpha Go / Hold Decision Checklist

Purpose: decide whether SabahLot can move from Controlled Alpha to wider limited release.

## Minimum Alpha Gate

| Gate | Requirement | Status |
|---|---|---|
| User Count | Minimum 10 selected users invited | Pending |
| Actual Testing | Majority users tested main functions | Pending |
| Critical Issues | No open Critical issue | Pending |
| Major Issues | Major issues reviewed and controlled | Pending |
| Save / Load | Stable after browser refresh | Pending |
| Import | KML / GeoJSON / CSV tested | Pending |
| Export | GeoJSON / KML / CSV tested | Pending |
| Mobile | Usable on phone screen | Pending |
| Disclaimer | Users understand output is preliminary | Pending |
| Privacy | Sensitive data private-by-default | Pending |
| Manual Review | Boundary/NCR/waris/dispute risk understood | Pending |

## Go / Hold / No-Go Decision

| Decision | Condition |
|---|---|
| Go Limited Release | No Critical issue, Major issues controlled, users understand preliminary output |
| Hold and Fix | Major issue affects core workflow, but fix is practical |
| No-Go | Critical issue, data loss, privacy risk, or serious misunderstanding of boundary/legal status |

## Non-Negotiable Safety Rule

Do not release SabahLot publicly if users misunderstand the output as official survey plan, certified boundary, ownership proof, subdivision approval, or land title status confirmation.
